# 论文排版工具 - 界面升级说明

## 当前界面定位

本次 GUI 升级不再把界面当成宣传页，而是按 **Windows 桌面参数配置工具** 来设计：
- 中文界面
- 纯文字按钮与导航
- 学术、稳重、实用
- 优先保证配置扫描效率、执行操作清晰和日志反馈明确

当前 GUI 固定使用 `sandstone` 主题，以保证样式稳定一致。

## 已完成的界面改造

### 1. 顶部标题区收紧
- 顶部只保留工具名称、定位说明和关键状态信息
- 不再让标题区占据主体高度
- 已移除主题切换入口，避免切换后样式缺失

### 2. 左侧配置导航重做
- 采用独立滚动容器
- 分类顺序调整为更符合填写流程：页面 -> 封面声明 -> 正文 -> 标题 -> 页眉页码 -> 目录参考 -> 图表
- 当前选中项有更明确的文字按钮状态

### 3. 主内容区可读性提升
- 表单按分组面板展示
- 标题、说明、参数区层级更清晰
- 输入框、增减按钮、下拉框统一为同一套工具样式
- 面板内容过长时可独立滚动

### 4. 底部操作区常驻
- 配置管理、文件与输出、执行操作、运行日志固定在底部区域
- 去掉了“快捷操作”之类容易分散注意力的入口
- 主按钮只保留“开始格式化”，权重更明确

### 5. 运行日志区优化
- 日志区与主配置分离
- 字体改为更适合中文界面的 `Microsoft YaHei UI`
- 增加日志内边距与行距，降低模糊和拥挤感

### 6. 滚动行为修复
- 左侧导航可直接滚轮滚动
- 右侧主配置区可直接滚轮滚动
- 滚轮按鼠标当前所在区域分发，不会左侧右侧串滚
- 滚动灵敏度已压低，避免一次滚过多内容

## 当前界面使用方式

### 启动 GUI

```bash
python run_gui.py
```

### 基本流程

1. 在左侧“配置导航”中逐项检查参数
2. 在底部选择输入文件和输出文件
3. 视需要勾选“跳过目录生成”
4. 点击“开始格式化”
5. 在运行日志区查看进度、警告和结果

## 当前界面的设计取舍

### 为什么不再支持多主题切换

此前切换 `ttkbootstrap` 不同主题后，会出现：
- tk / ttk 混合控件样式不一致
- 按钮、边框、分组框局部退回默认风格
- 同一页面不同组件观感割裂

因此当前版本选择：
- 固定主题
- 把精力集中在布局、层级、滚动、日志和按钮权重上
- 先保证这一个默认主题足够稳定可用

## 后续可继续优化的方向

- 进一步压缩顶部信息区高度
- 优化高 DPI 屏幕上的控件边框细节
- 针对更复杂的封面字段场景补充更紧凑的编辑布局
- 为日志区增加导出或复制能力
